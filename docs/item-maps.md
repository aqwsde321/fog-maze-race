# 아이템 맵 설계 초안

상태: `planned`
작성일: `2026-04-07`
대상 브랜치: `codex/item-maps`

이 문서는 아직 구현되지 않은 아이템 맵 규칙의 초안이다. 현재 배포 버전에는 아이템 박스,
인벤토리, 함정 사용이 포함되지 않는다. 목적은 구현 전에 규칙, 데이터 모델, 테스트 기준을
먼저 고정하는 것이다.

## 목표

- 일반 레이스 규칙은 유지하면서, 일부 맵에서만 아이템 박스 상호작용을 추가한다.
- 첫 구현은 복잡한 공격 스킬이나 다중 인벤토리 없이 `1칸 인벤토리 + 단일 함정`만 다룬다.
- authoritative 판정은 전부 서버가 담당하고, 클라이언트는 렌더링과 사용 입력만 담당한다.

## 1차 범위

- 아이템 맵 여부를 맵 단위로 켜고 끈다.
- 레이스 시작 시 `active racers` 수만큼 랜덤 박스를 생성한다.
- 플레이어 인벤토리는 정확히 1칸이다.
- 아이템을 들고 있는 동안에는 다른 랜덤 박스를 줍지 못한다.
- 첫 아이템 타입은 `ice_trap` 하나만 지원한다.
- `ice_trap`은 설치형 함정이고, 밟은 상대를 잠시 얼린다.

## 제외 범위

- 다중 인벤토리
- 드래그형 조준, 투사체, 텔레포트
- 여러 아이템 동시 지원
- 아이템 상점, 점수형 업그레이드
- 아이템 전용 봇 전략

## 사용자 경험

### 아이템 맵

- 일반 맵과 별도로, 맵 정의에서 `아이템 박스 활성화` 여부를 가진다.
- 이 설정은 `방 모드(normal / bot_race)` 와는 별개다.
- 즉 `일반 방인지`, `봇 전용 방인지`를 고르는 것과 다르게, 특정 맵 자체가 `아이템 맵`
  인지 아닌지를 정한다.
- 1차 구현에서는 방을 만들 때 아이템 ON/OFF를 따로 고르지 않고, 선택된 맵이 아이템 맵일
  때만 아이템 규칙을 켠다.
- 아이템 맵으로 시작한 라운드만 랜덤 박스가 생성된다.
- 일반 맵은 현재 규칙을 그대로 유지한다.

### 인벤토리

- 플레이어는 한 번에 하나의 아이템만 들 수 있다.
- 인벤토리가 비어 있을 때만 박스를 획득한다.
- 이미 아이템을 들고 있으면 박스 위를 지나가도 획득하지 않는다.
- 인벤토리 상태는 HUD에 작은 1칸 슬롯으로 표시한다.

### 사용 흐름

- 플레이어가 아이템을 사용하면 인벤토리는 즉시 비워진다.
- 입력 방식은 `사용 버튼 + 단축키` 조합을 기본안으로 잡는다.
- 초기 권장 단축키는 `Space`다. 실제 적용 전 UX 점검이 필요하다.

## 아이템 규칙

### 랜덤 박스

- 생성 시점: 매치가 `playing`으로 전환되는 순간 한 번 생성
- 생성 개수: 그 라운드의 `active racers` 수와 동일
- 생성 위치:
  - `mazeZone` 안의 이동 가능한 타일만 허용
  - 즉 실제로 플레이어가 걸어 다니는 `길 타일` 중에서 랜덤 생성 가능
  - `startZone`, `connector`, `goalZone`, `fakeGoalTiles` 제외
  - 한 타일에 박스 1개만 허용
  - 가능한 한 시작 진입 직후 구간과 골 직전 구간은 피한다
- 재생성: 1차에서는 없음

예시:

- 가능: 미로 안 통로, 교차로, 우회로 같은 `walkable floor`
- 불가: 시작 구역, 세로 연결 통로, 골 타일, 가짜 골 타일, 벽 타일

### ice_trap

- 획득 시 인벤토리에 `ice_trap` 저장
- 사용 시 현재 플레이어 위치에 `ice_trap`을 설치
- 설치 즉시 발동하지 않고, 설치자가 해당 타일을 벗어난 뒤 `armed` 상태가 된다
- `armed` 상태에서 다른 레이서가 진입하면 발동한다
- 발동 효과:
  - 대상 플레이어는 `1500ms` 동안 이동 불가
  - 빙결 중에는 이동 입력을 받아도 서버가 거절한다
  - 함정은 1회 발동 후 제거된다
- 설치자 본인은 자신의 함정에 걸리지 않는다
- 라운드 종료 시 남은 함정은 전부 제거된다

## 시각 콘셉트

### 랜덤 박스

- 정사각형 상자 위에 `?` 대신 눈송이 문양을 얹은 차가운 청록색 박스
- 기본 팔레트는 `cyan / ice blue / pale white`
- 멀리서도 바닥 타일과 구분되도록 외곽선과 작은 발광 효과를 둔다

### ice_trap

- 바닥 위에 얇은 얼음 균열과 눈송이 모양이 겹쳐 보이는 디칼
- 색감은 `#9BE7FF`, `#D9F7FF`, `rgba(180, 240, 255, 0.45)` 계열
- 발동 순간에는 짧은 서리 링과 얼음 파편 스플래시를 재생한다
- 빙결된 플레이어 마커 아래에는 짧게 서리 그림자를 추가한다

## 도메인 제안

### MapDefinition

기존 맵 정의에 아이템 활성화 플래그를 추가한다.

```ts
type MapFeatureFlags = {
  fakeGoals?: boolean;
  itemBoxes?: boolean;
};
```

### Match

매치는 현재 라운드의 박스와 함정을 authoritative 상태로 가진다.

```ts
type MatchItemBox = {
  boxId: string;
  position: GridPosition;
  itemType: "ice_trap";
  state: "available" | "claimed";
};

type MatchTrap = {
  trapId: string;
  ownerPlayerId: string;
  position: GridPosition;
  state: "arming" | "armed" | "triggered";
  armedAt: number | null;
};
```

### RoomMember

방 멤버는 1칸 인벤토리와 빙결 상태를 가진다.

```ts
type RoomMemberInventory = {
  itemType: "ice_trap" | null;
};

type StatusEffect = {
  frozenUntil: number | null;
};
```

## 서버 규칙

- 박스 생성, 획득, 사용, 발동 판정은 전부 서버가 처리한다.
- `frozenUntil > now` 인 플레이어의 이동은 authoritative하게 거절한다.
- 아이템 사용은 `playing` 상태의 `racer`만 가능하다.
- `spectator`는 박스를 줍거나 사용할 수 없다.
- 박스 생성과 함정 설치는 스냅샷/이벤트로 모든 클라이언트에 동기화한다.

## 클라이언트 표시

- 보드 위에 `item box`와 `ice_trap`를 별도 타일 오버레이로 렌더링한다.
- 인벤토리 슬롯은 우측 패널 또는 하단 HUD에 1칸만 표시한다.
- 빙결 상태일 때 플레이어 주변에 얼음 효과와 남은 시간 피드백을 짧게 보여준다.
- 시작 전 프리뷰(`waiting`)에는 아이템 오버레이를 렌더하지 않는다.

## 봇 정책

1차 구현에서는 아이템 authoritative 동작을 먼저 완성한다.

- 봇도 박스를 획득할 수는 있어야 한다.
- 다만 `언제 사용할지`는 별도 정책이 필요하다.
- 추천 후속 순서:
  1. `join` 봇: 박스를 들고 있으면 일정 주기마다 단순 설치
  2. `explore` 봇: 최근 지나온 교차로 또는 추적당하는 구간에서 설치

## 테스트 기준

### 계약 테스트

- 아이템 맵 라운드 시작 시 레이서 수와 같은 개수의 박스가 생성된다.
- 박스는 금지 구역이 아닌 유일한 타일에만 생성된다.
- 인벤토리가 찬 플레이어는 두 번째 박스를 획득하지 못한다.
- 관전자는 박스를 줍지 못하고 사용할 수도 없다.

### 도메인 테스트

- `ice_trap` 사용 후 설치자가 타일을 벗어나기 전에는 발동하지 않는다.
- 다른 레이서가 진입하면 대상만 `frozen` 상태가 된다.
- 빙결 시간 동안 이동이 거절되고, 시간이 지나면 다시 이동할 수 있다.
- 라운드 종료 시 남은 박스와 함정이 초기화된다.

### 웹 테스트

- 아이템 맵에서는 박스와 함정 오버레이가 렌더된다.
- 일반 맵에서는 아이템 UI가 노출되지 않는다.
- 인벤토리 슬롯 1칸이 정확히 반영된다.
- 빙결 중 HUD 피드백이 나타난다.

## 구현 순서 제안

1. 공유 계약과 도메인 타입 추가
2. 서버 매치 상태에 박스/함정 authoritative 로직 추가
3. 스냅샷과 이벤트 동기화 추가
4. 웹 오버레이와 인벤토리 HUD 추가
5. 플레이어 입력으로 아이템 사용 추가
6. 마지막에 봇 사용 정책 추가

## 열린 결정 사항

- 아이템 사용 단축키를 `Space`로 확정할지
- 아이템 활성화는 1차에서 `맵 플래그만`으로 확정할지
- `ice_trap` 빙결 시간을 `1500ms`로 확정할지
- 박스 스폰 금지 반경을 시작/골 주변 어디까지 둘지
