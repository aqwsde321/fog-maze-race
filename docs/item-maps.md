# 아이템 맵 가이드

현재 아이템 맵은 `맵 플래그`로만 활성화됩니다. 즉, 방 생성 시 별도 옵션을 켜는 방식이 아니라, 아이템이 켜진 맵에 걸렸을 때만 아이템 규칙이 적용됩니다.

1차 구현 기준으로는 기본 제공 맵 중 `kappa-trap`이 아이템 맵입니다.

## 현재 규칙

- 인벤토리는 `1칸`만 지원합니다.
- 아이템을 들고 있는 동안에는 다른 박스를 추가로 먹을 수 없습니다.
- 기본 생성 규칙은 `레이서 수 x 2`입니다.
- 관리자 맵 편집에서 생성 수 규칙을 바꿀 수 있습니다.
  - `참가자 수 배수`: 레이서 수에 배수를 곱해 생성
  - `고정 개수`: 항상 같은 개수 생성
- 랜덤 박스는 `mazeZone` 안의 실제 길 타일에서만 생성됩니다.
- `startZone`, `connector`, `goalZone`, `fakeGoalTiles`, 시작 진입부 일부, 골 주변 일부는 생성 제외 구간입니다.
- 첫 아이템은 `ice_trap` 하나만 지원합니다.
- 아이템 사용 키는 `Space`입니다.

## 얼음 함정

`ice_trap`을 사용하면 현재 칸에 얼음 함정을 설치합니다.

- 설치 직후 상태는 `arming`
- 설치한 플레이어가 그 칸을 벗어나면 `armed`
- 다른 레이서가 해당 칸에 들어오면 발동
- 발동한 플레이어는 `1500ms` 동안 얼어붙어 이동할 수 없습니다.
- 설치한 플레이어 본인은 자기 함정에 걸리지 않습니다.

## UI 표시

- 플레이 중 아이템을 들고 있으면 좌측 상단 HUD에 `얼음 함정`과 `Space 사용`이 표시됩니다.
- 얼어붙은 상태면 별도 상태 카드가 표시됩니다.
- 맵 위에는 랜덤 박스와 얼음 함정이 오버레이로 렌더됩니다.

## 봇 동작

- 탐험형/빠른 봇 모두 아이템 박스를 주울 수 있습니다.
- 봇이 `ice_trap`을 들고 `mazeZone` 안에 있으면, 현재 칸에 기존 함정이 없을 때 자동으로 설치합니다.
- 설치 후 다음 이동에서 함정이 `armed` 상태가 됩니다.

## 구현 메모

- 서버 authoritative 규칙입니다.
- 아이템 보유, 얼음 상태, 랜덤 박스, 함정 상태는 모두 스냅샷에 포함됩니다.
- 현재 1차 범위에는 추가 아이템, 인벤토리 다중 슬롯, 수동 스폰 포인트 편집은 포함하지 않습니다.
